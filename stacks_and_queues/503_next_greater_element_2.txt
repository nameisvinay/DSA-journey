### ğŸ§  Key Insight:

ğŸ’¡ Since the array is **circular**, simulate two passes (2 \* n) to handle wrap-around.
ğŸ’¡ We still use a **monotonic decreasing stack** to track **future greater elements**.
ğŸ’¡ Always `stack.pop()` smaller/equal elements.
ğŸ’¡ Use `i % n` to simulate circular indexing.

### âœ… Final Code (Python):

```python
class Solution:
    def nextGreaterElements(self, nums: List[int]) -> List[int]:
        n = len(nums)
        res = [-1] * n
        stack = []

        for i in range(2 * n - 1, -1, -1):
            curr = nums[i % n]
            while stack and stack[-1] <= curr:
                stack.pop()

            if i < n:
                res[i] = stack[-1] if stack else -1

            stack.append(curr)

        return res
```
